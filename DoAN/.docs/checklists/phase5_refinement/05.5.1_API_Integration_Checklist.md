# 05.5.1 API Integration (API Integration & Mock Elimination Checklist)

Chiến dịch "Săn lùng & Tiêu diệt" dữ liệu giả (Mock Data) nhằm kết nối hoàn thiện đồng bộ dữ liệu thời gian thực giữa hai cổng Frontend và Backend API.

---

## 📋 Danh sách các đầu việc thực thi (Checklist)

- [x] **Bước 1: Quét sạch dữ liệu giả toàn cục (Global Mock Elimination)**
  - Thực hiện tìm kiếm toàn cục (Global Search) trong mã nguồn dự án các từ khóa: `MOCK_`, `mockProducts`, `dummyData`, `dummyOrders`, hoặc các mảng khai báo dữ liệu cứng trực tiếp trong React State.
  - Tiến hành dọn dẹp và xóa bỏ hoàn toàn để chuẩn bị kết nối dữ liệu sống.

- [x] **Bước 2: Tích hợp API Xác thực (Authentication API)**
  - Kết nối API Đăng nhập và Đăng ký tài khoản (`POST /api/auth/login`, `POST /api/auth/register`) thông qua `api-gateway`.
  - Phía Client: Lưu trữ mã token JWT nhận được vào `LocalStorage` hoặc cấu hình Backend trả về `HttpOnly Cookie` an toàn để bảo mật tuyệt đối chống tấn công XSS.

- [x] **Bước 3: Tích hợp API Sản phẩm & Danh mục (Product Catalog API)**
  - Thay thế dữ liệu hiển thị tại trang chủ, trang danh mục và trang chi tiết sản phẩm bằng cách gọi API thật `GET /api/products` và `GET /api/categories`.
  - Hỗ trợ đầy đủ các tham số lọc theo danh mục, tìm kiếm và phân trang y tế.

- [x] **Bước 4: Tích hợp API Giỏ hàng & Thanh toán (Cart & Order APIs)**
  - Nối API quản lý giỏ hàng trực tuyến (nếu đồng bộ database) hoặc API tạo đơn hàng mới `POST /api/orders` chuyển tiếp mảng items từ giỏ hàng LocalStorage kèm thuộc tính `prescriptionId` khi có thuốc Rx.

- [x] **Bước 5: Tích hợp Luồng Nghiệp vụ Toa thuốc Rx (Rx Prescription Workflow)**
  - **Storefront:** Nối API gửi ảnh toa đơn `POST /api/prescriptions` (sử dụng `FormData` upload lên Cloudinary).
  - **Admin Portal:** Nối API lấy danh sách đơn chờ duyệt `GET /api/prescriptions?status=PENDING` và API gửi báo giá bốc thuốc y khoa `PATCH /api/prescriptions/:id/approve`.

---

## 🧪 HOẠT ĐỘNG KIỂM THỬ (Verification Steps)

> [!NOTE]
> ### 🔍 Quy trình Kiểm tra
> 1. Tắt tạm thời server Backend, mở trình duyệt truy cập Storefront và Admin Portal ➡️ Kiểm tra xem hệ thống có hiển thị các màn hình thông báo thân thiện (Empty States UI/Loading Spinners) hay bị crash trắng màn hình.
> 2. Bật lại Server Backend, thực hiện kiểm duyệt luồng mua sắm sản phẩm OTC và Rx, chốt đơn để đảm bảo dữ liệu hiển thị đồng bộ 100% thời gian thực.
>
> ### 🎯 Tiêu chí Pass
> - Không còn bất kỳ dữ liệu Mock/Dummy tĩnh nào xuất hiện trên giao diện.
> - Các Component hiển thị mượt mà, có cơ chế bắt lỗi mất mạng hoặc mảng dữ liệu rỗng trả về từ API (không bị crash hay vỡ giao diện).
