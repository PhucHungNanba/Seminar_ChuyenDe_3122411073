# 05.5.2 Optimization & Code Cleanup (Performance & Quality Checklist)

Checklist hướng dẫn dọn dẹp các tài nguyên dư thừa, tối ưu hóa băng thông tải ảnh y khoa đám mây, xử lý lỗi tập trung và cấu hình nén đóng gói mã nguồn đạt hiệu năng cao nhất.

---

## 📋 Danh sách các đầu việc thực thi (Checklist)

- [x] **Bước 1: Tối ưu hóa hiển thị hình ảnh từ Cloudinary**
  - Rà soát toàn bộ các thẻ `<img src={...} />` render ảnh sản phẩm hoặc ảnh toa thuốc lấy từ Cloudinary CDN.
  - Cấu hình chèn chuỗi tham số nén tự động `/q_auto,f_auto/` vào giữa link URL để Cloudinary tự động chuyển đổi định dạng ảnh nhẹ nhất (như WebP/AVIF) và nén dung lượng phù hợp mà vẫn giữ độ nét chữ viết tay đơn thuốc.

- [x] **Bước 2: Dọn dẹp Code rác & Thư viện dư thừa (Clean Code)**
  - Thực hiện kiểm tra tệp `package.json` ở cả Frontend, Admin và Backend, xóa bỏ toàn bộ các package cài đặt thử nghiệm không sử dụng.
  - Xóa bỏ các file component nháp, file test tạm thời hoặc tệp tin log trong quá trình làm việc.
  - Tiến hành quét và xóa bỏ triệt để tất cả các dòng lệnh `console.log` để bảo mật thông tin trên môi trường Production.

- [x] **Bước 3: Hoàn thiện Cơ chế Bắt lỗi Tập trung UI (Error Handling UI)**
  - Rà soát toàn bộ các hàm gọi API Axios (Zustand/Components).
  - Đảm bảo trong khối `catch (error)`, mọi lỗi phản hồi từ API (như lỗi kết nối, mất mạng, quá tải server) đều được tiếp nhận và hiển thị lên màn hình dưới dạng thông báo Toast thân thiện với người dùng (Ví dụ: *"⚠️ Không thể kết nối đến máy chủ, vui lòng kiểm tra mạng và thử lại sau"*).

## 🧪 HOẠT ĐỘNG KIỂM THỬ (Verification Steps)

> [!NOTE]
> ### 🔍 Quy trình Kiểm tra
> 1. Mở Terminal tại Frontend và Admin Portal, thực thi lệnh build chính thức:
>    ```bash
>    npm run build
>    ```
>
> ### 🎯 Tiêu chí Pass
> - Tiết trình biên dịch (Compilation) hoàn tất 100% thành công, không phát sinh bất kỳ cảnh báo lỗi kiểu (TypeScript Compilation Errors) nào.
> - Kết xuất thư mục đóng gói `/dist` gọn gàng, dung lượng các tệp tin bundle nhỏ gọn nhờ được chunking code chuẩn xác.
