# 05.3.2 Admin Layout & Routing (Back-office UI Setup Checklist)

Checklist hướng dẫn thiết lập hệ thống định tuyến, giao diện khung sườn phân mảnh (Sidebar/Header/AdminLayout) và các trang tĩnh quản trị cho Dược sĩ và Admin tại Admin Portal.

---

## 📋 Danh sách các đầu việc thực thi (Checklist)

- [x] **Bước 1: Thiết lập thư viện nền tảng**
  - Cài đặt `react-router-dom` và `lucide-react` trong project `/Admin`.

- [x] **Bước 2: Xây dựng Giao diện Sidebar điều hướng bên trái `Sidebar.tsx`**
  - Tạo file `src/components/layout/Sidebar.tsx` với các thông số:
    - Logo tiêu đề: `"PharmaAdmin"` sắc nét.
    - Avatar & tên định danh người đăng nhập giả lập (Ví dụ: `Dược sĩ Nguyễn Văn A`).
    - Menu quản trị gồm các liên kết:
      - **Tổng quan** (Icon `LayoutDashboard`) ➡️ Trỏ tới `/admin`
      - **Xét duyệt Đơn thuốc** (Icon `ClipboardList` đại diện cho Rx) ➡️ Trỏ tới `/admin/rx-approval`
      - **Quản lý Kho** (Icon `Package`) ➡️ Trỏ tới `/admin/inventory`
      - **Quản lý Người dùng** (Icon `Users`) ➡️ Trỏ tới `/admin/users`
    - Phong cách: Nền xanh Navy tối sang trọng, các menu item đang `active` được highlight đổi màu nền sang xanh sáng hoặc thêm đường viền vàng nổi bật.

- [x] **Bước 3: Xây dựng Giao diện Header trên cùng `Header.tsx`**
  - Tạo file `src/components/layout/Header.tsx`.
  - Chứa ô tìm kiếm nghiệp vụ nhanh ở giữa và Icon `Bell` (Chuông thông báo real-time của đơn thuốc mới) ở góc phải.

- [x] **Bước 4: Xây dựng Layout Quản trị chính `AdminLayout.tsx`**
  - Tạo file `src/components/layout/AdminLayout.tsx`.
  - Bố cục màn hình: Chia làm 2 cột:
    - Cột trái: `Sidebar` cố định với chiều rộng `250px`.
    - Cột phải: Khối chứa `Header` phía trên, phía dưới là vùng render `<Outlet />` có thanh scroll cuộn nội dung độc lập.

- [x] **Bước 5: Tạo các Trang Placeholder Quản trị rỗng**
  - Tạo thư mục `src/pages/` và khởi tạo các trang trống hiển thị các tiêu đề H1 to, rõ màu sắc riêng biệt để dễ đối chiếu:
    - [x] `DashboardPage.tsx`
    - [x] `RxApprovalPage.tsx` (Trình duyệt đơn thuốc y khoa Split-screen)
    - [x] `InventoryPage.tsx` (Quản lý kho hàng & số lượng tồn chi nhánh)
    - [x] `UserManagementPage.tsx` (Quản trị tài khoản & RBAC)

- [x] **Bước 6: Cấu hình Định tuyến `router.tsx` & Redirect**
  - Tạo file `src/routes/router.tsx` sử dụng `createBrowserRouter`.
  - [x] Khai báo các đường dẫn `/admin`, `/admin/rx-approval`, `/admin/inventory`, `/admin/users` trỏ đúng vào các trang rỗng vừa tạo, bọc dưới `AdminLayout`.
  - [x] Thiết lập cấu hình Redirect: Nếu người dùng truy cập trực tiếp vào gốc `/`, hệ thống tự động redirect sang trang tổng quan `/admin`.
  - Kết nối router vào tệp tin chạy chính `src/main.tsx`.

## 🧪 HOẠT ĐỘNG KIỂM THỬ (Verification Steps)

> [!NOTE]
> ### 🔍 Quy trình Kiểm tra
> 1. Chạy ứng dụng Admin Portal: `npm run dev` trên cổng `5174`.
> 2. Kích hoạt `Browser Subagent` truy cập `http://localhost:5174`.
> 3. Click lần lượt qua tất cả các liên kết trong Menu Sidebar ở cột bên trái để kiểm tra phản hồi render nội dung ở vùng hiển thị cột bên phải.
> 4. Thực hiện co giãn kích thước màn hình máy tính giả lập để kiểm thử độ co giãn tự nhiên (Responsive) của Sidebar và Header.
>
> ### 🎯 Tiêu chí Pass
> - Nội dung vùng `<Outlet />` bên phải thay đổi chính xác theo các trang tương ứng khi click Menu Sidebar bên trái.
> - Luồng chuyển đổi trang diễn ra tức thì, không vấp lỗi hay crash màn hình.
> - Layout Sidebar cố định vững chãi, phần hiển thị Header và trang nội dung co giãn tốt khi thay đổi kích thước màn hình mà không bị tràn hoặc đè lên nhau.
