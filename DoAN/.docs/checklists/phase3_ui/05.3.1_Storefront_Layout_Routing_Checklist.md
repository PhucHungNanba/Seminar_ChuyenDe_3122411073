# 05.3.1 Storefront Layout & Routing (Client UI Setup Checklist)

Checklist hướng dẫn thiết lập hệ thống định tuyến, giao diện khung sườn (Navbar/Footer/MainLayout) và các trang tĩnh rỗng cho phía Khách hàng (Storefront) để đảm bảo trải nghiệm mua sắm nhanh gọn, trực quan.

---

## 📋 Danh sách các đầu việc thực thi (Checklist)

- [x] **Bước 1: Thiết lập thư viện nền tảng**
  - Cài đặt `react-router-dom` và `lucide-react` trong project `/Frontend`.

- [x] **Bước 2: Xây dựng Giao diện Điều hướng Trên cùng `Navbar.tsx`**
  - Tạo file `src/components/layout/Navbar.tsx` với các yêu cầu:
    - Logo thương hiệu: Text `"PharmaCare"` nổi bật.
    - Thanh tìm kiếm (Mock UI): Một ô input tìm kiếm sản phẩm bo góc tròn kèm kính lúp.
    - Icon Giỏ hàng: Icon `ShoppingCart` kèm vòng tròn Badge số lượng giả màu vàng `#FFD700`.
    - Nút Đăng nhập / Profile.
    - Phong cách: Nền xanh Navy `#003580`, chữ màu trắng, bố cục Flexbox giãn cách đều và Responsive.

- [x] **Bước 3: Xây dựng Giao diện Chân trang `Footer.tsx`**
  - Tạo file `src/components/layout/Footer.tsx`.
  - Chứa các thông tin tĩnh phân nhóm: Về chúng tôi, Chính sách nhà thuốc, Hướng dẫn mua hàng, Hotline CSKH 24/7.

- [x] **Bước 4: Xây dựng Layout bọc chính `MainLayout.tsx`**
  - Tạo file `src/components/layout/MainLayout.tsx`.
  - Bố cục: Bọc `Navbar` phía trên, thẻ `<Outlet />` ở giữa nhận diện nội dung render, và `Footer` cố định bên dưới.

- [x] **Bước 5: Tạo các Trang Placeholder rỗng**
  - Tạo thư mục `src/pages/` và khởi tạo các trang tĩnh có background màu nhạt kèm chữ to ở giữa màn hình để dễ dàng phân biệt:
    - [x] `HomePage.tsx` (Trang chủ)
    - [x] `ProductCatalogPage.tsx` (Danh mục thuốc & thực phẩm chức năng)
    - [x] `MedicineDetailPage.tsx` (Chi tiết sản phẩm thuốc)
    - [x] `CartPage.tsx` (Giỏ hàng và kiểm soát Rx)
    - [x] `ProfilePage.tsx` (Hồ sơ người dùng & Sổ lưu đơn thuốc Prescription Vault)

- [x] **Bước 6: Khai báo Cấu trúc Định tuyến hiện đại `router.tsx`**
  - Tạo file `src/routes/router.tsx` sử dụng hàm `createBrowserRouter`.
  - Khai báo các đường dẫn (`/`, `/products`, `/products/:id`, `/cart`, `/profile`) trỏ tương ứng vào các trang placeholder vừa tạo, bọc tất cả dưới `MainLayout`.
  - Gắn `<RouterProvider router={router} />` vào tệp chạy chính `src/main.tsx`.

## 🧪 HOẠT ĐỘNG KIỂM THỬ (Verification Steps)

> [!NOTE]
> ### 🔍 Quy trình Kiểm tra
> 1. Chạy ứng dụng Frontend: `npm run dev` trên cổng `5173`.
> 2. Kích hoạt `Browser Subagent` truy cập `http://localhost:5173`.
> 3. Click thử vào nút Logo để quay về trang chủ (`/`), click vào Icon Giỏ hàng để chuyển hướng sang trang giỏ hàng (`/cart`), và click vào các liên kết khác để thử nghiệm định tuyến.
>
> ### 🎯 Tiêu chí Pass
> - Ứng dụng điều hướng trơn tru qua lại giữa các trang mà không bị hiện tượng trắng màn hình (White Screen) hay mất kết nối.
> - Layout của `Navbar` và `Footer` được bảo toàn nguyên vẹn ở tất cả các trang.
