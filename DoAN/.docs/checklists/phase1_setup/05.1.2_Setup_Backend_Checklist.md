# 05.1.2 Setup Backend Microservices (Scaffolding Checklist)

Checklist thực thi việc khởi tạo và thiết lập khung xương cho 5 tệp tin dịch vụ backend độc lập phục vụ kiến trúc Microservices phân rã.

---

## 📋 Danh sách các đầu việc thực thi (Checklist)

- [x] **Bước 1: Khởi tạo thư mục dự án cho các Microservices**
  - Tạo cấu trúc thư mục con cho từng microservice tại `/Backend`:
    - `/Backend/api-gateway`
    - `/Backend/user-service`
    - `/Backend/product-service`
    - `/Backend/order-service`
    - `/Backend/ai-service` (Python FastAPI)

- [x] **Bước 2: Cài đặt package.json & dependencies cho các Node services**
  - Thực thi khởi tạo Node.js project và cài đặt dependencies cho các Express services (`api-gateway`, `user-service`, `product-service`, `order-service`):
    ```bash
    npm install express mongoose cors dotenv jsonwebtoken multer cloudinary
    ```

- [x] **Bước 3: Cấu hình môi trường TypeScript & Dev Tools**
  - Cài đặt `typescript`, `ts-node`, `nodemon` và các file `@types/*` tương ứng vào devDependencies ở từng Node-based microservice.
  - Khởi tạo file cấu hình `tsconfig.json` chuẩn hóa (Strict Mode: `true`).

- [x] **Bước 4: Tạo cấu trúc thư mục phân tầng cho từng Microservice**
  - Thiết lập đầy đủ các thư mục con phục vụ kiến trúc Clean Architecture:
    - `/src/controllers/` (Tầng điều phối request)
    - `/src/models/` (Tầng Schema định nghĩa database)
    - `/src/routes/` (Tầng định tuyến Express)
    - `/src/middlewares/` (Tầng xử lý trung gian Auth/RBAC/Validation)
    - `/src/services/` (Tầng xử lý logic nghiệp vụ chính)

- [x] **Bước 5: Khởi tạo Python AI Service (ai-service)**
  - Khởi tạo cấu trúc dự án Python FastAPI với `main.py` chạy trên cổng `8000`, thiết lập file `requirements.txt` đầy đủ (`fastapi`, `uvicorn`, `pandas`, `mlxtend`...).
  - Thiết lập môi trường ảo Python `venv` và các router phục vụ giải thuật luật kết hợp.

- [x] **Bước 6: Xây dựng tệp máy chủ cơ bản `server.ts` & healthcheck routes**
  - Viết code Express thiết lập các cổng port exposed tương ứng:
    - `api-gateway`: Cổng `3000`
    - `user-service`: Cổng `3001`
    - `product-service`: Cổng `3002`
    - `order-service`: Cổng `3003`
  - Thiết lập các endpoint `/health` trả về kết quả kiểm tra sức khỏe của từng container y tế.

---

## 🧪 HOẠT ĐỘNG KIỂM THỬ (Verification Steps)

> [!NOTE]
> ### 🔍 Quy trình Kiểm tra
> 1. Khởi chạy độc lập các máy chủ phát triển của từng service.
> 2. Gửi yêu cầu HTTP kiểm tra sức khỏe tới `api-gateway` và các service nhánh.
>
> ### 🎯 Tiêu chí Pass
> - Tất cả 5 Microservices khởi chạy thành công không có lỗi crash.
> - API `GET http://localhost:3000/api/health` phản hồi mã trạng thái `HTTP 200 OK` cùng JSON chỉ ra các dịch vụ hoạt động khỏe mạnh.
