# 05.1.3 Setup Docker Virtualization (Containerization Checklist)

Checklist hướng dẫn thiết lập Dockerfiles và cấu hình Docker Compose để ảo hóa hoàn chỉnh môi trường phát triển của PharmaCare, đảm bảo tính đóng gói độc lập cho cả 2 frontend và 4 Express backend microservices.

---

## 📋 Danh sách các đầu việc thực thi (Checklist)

- [x] **Bước 1: Viết tệp Dockerfile cho Frontend (Storefront App)**
  - Tạo file `Frontend/Dockerfile` sử dụng base image `node:20-alpine` chạy ở cổng exposed `5173`.

- [x] **Bước 2: Viết tệp Dockerfile cho Admin Portal App**
  - Tạo file `Admin/Dockerfile` sử dụng base image `node:20-alpine` chạy ở cổng exposed `5174`.

- [x] **Bước 3: Viết các tệp Dockerfile cho 4 Express Backend Microservices**
  - Tạo file `Dockerfile` cho từng service: `api-gateway`, `user-service`, `product-service`, và `order-service` sử dụng `node:20-alpine`.

- [x] **Bước 4: Thiết lập các tệp tin loại bỏ `.dockerignore`**
  - Viết file `.dockerignore` tại các thư mục con nhằm loại bỏ `node_modules`, `dist/` và `.env` cục bộ để tối ưu hóa quá trình đóng gói và tránh ghi đè thư viện từ máy host vào container.

- [x] **Bước 5: Thiết lập tệp tin điều phối toàn cục `docker-compose.yml`**
  - Viết file `docker-compose.yml` tại thư mục gốc của dự án.
  - Khai báo và cấu hình đồng bộ 6 service chính chạy chung mạng ảo nội bộ (`medicine-network` bridge network):
    - `api-gateway` (Cổng `3000:3000`)
    - `user-service` (Cổng `3001:3001`)
    - `product-service` (Cổng `3002:3002`)
    - `order-service` (Cổng `3003:3003`)
    - `frontend` (Cổng `5173:5173`)
    - `admin` (Cổng `5174:5174`)

---

## 🧪 HOẠT ĐỘNG KIỂM THỬ (Verification Steps)

> [!NOTE]
> ### 🔍 Quy trình Kiểm tra
> 1. Khởi chạy toàn hệ thống thông qua lệnh:
>    ```bash
>    docker compose up -d --build
>    ```
> 2. Chạy lệnh kiểm tra trạng thái hoạt động của các Container:
>    ```bash
>    docker ps
>    ```
> 3. Kiểm duyệt các cổng của các container hoạt động bình thường.
>
> ### 🎯 Tiêu chí Pass
> - Toàn bộ các Container khởi động và giữ trạng thái `Up` (Running) khỏe mạnh không tự crash.
> - Giao diện Storefront, Admin và Backend API phản hồi đầy đủ và chuẩn xác dữ liệu tại các địa chỉ Port exposed công khai.
