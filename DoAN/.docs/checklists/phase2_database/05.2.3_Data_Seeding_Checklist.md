# 05.2.3 Data Seeding (Mock Data Population Checklist)

Checklist hướng dẫn lập trình bộ script nạp tự động dữ liệu sản phẩm mẫu từ file JSON gốc dự án vào MongoDB Atlas phục vụ công tác phát triển và kiểm thử giao diện.

---

## 📋 Danh sách các đầu việc thực thi (Checklist)

- [x] **Bước 1: Khởi tạo thư mục Seeder**
  - Tạo thư mục `src/seeders/` tại `product-service`.

- [x] **Bước 2: Viết script nạp dữ liệu `seedProducts.ts`**
  - Tạo file `src/seeders/seedProducts.ts`.
  - Import `fs` để đọc file dữ liệu mồi y tế `db_products.products.json` từ thư mục gốc dự án.
  - Sử dụng kết nối `db_products` để tương tác trực tiếp với database.

- [x] **Bước 3: Lập trình Logic Seeding an toàn**
  - Thực hiện tuần tự các bước:
    1. Xóa sạch dữ liệu cũ trong collection `products` bằng `ProductModel.deleteMany({})`.
    2. Chuyển đổi file dữ liệu JSON đọc được sang mảng đối tượng JS.
    3. Import hàng loạt dữ liệu y tế vào collection qua `ProductModel.insertMany(productsData)`.
    4. Thêm log thông báo số lượng bản ghi nạp thành công bằng Tiếng Việt.
    5. Đóng kết nối database an toàn.

- [x] **Bước 4: Cấu hình Script chạy tự động trong `package.json`**
  - Thêm câu lệnh chạy script vào mục `"scripts"` của `product-service` `package.json`:
    ```json
    "seed": "ts-node src/seeders/seedProducts.ts"
    ```

---

## 🧪 HOẠT ĐỘNG KIỂM THỬ (Verification Steps)

> [!NOTE]
> ### 🔍 Quy trình Kiểm tra
> 1. Mở Terminal tại `product-service` và chạy lệnh:
>    ```bash
>    npm run seed
>    ```
> 2. Mở công cụ MongoDB Compass, kết nối và truy cập vào database `db_products`.
>
> ### 🎯 Tiêu chí Pass
> - Script chạy hoàn tất không vấp lỗi kết nối hay lỗi định dạng dữ liệu.
> - Collection `products` trong `db_products` được khởi tạo thành công và chứa đầy đủ dữ liệu mồi tương ứng với tệp JSON gốc.
