# 05.2.1 Database Connection (Multi-DB Connectivity Checklist)

Checklist hướng dẫn thiết lập 3 kết nối Mongoose độc lập tương ứng tại 3 microservices khác nhau tới 3 Database logical khác nhau để hoàn thiện nền móng phân tách dữ liệu.

---

## 📋 Danh sách các đầu việc thực thi (Checklist)

- [x] **Bước 1: Khai báo biến môi trường kết nối**
  - Mở tệp `.env` trong từng microservice và cấu hình đường dẫn liên kết MongoDB Atlas:
    ```env
    MONGO_URI=mongodb+srv://hoi321668_db_user:ppessWAOiuSL2lys@medicine.g3veeqn.mongodb.net/?appName=medicine
    ```

- [x] **Bước 2: Tạo tệp cấu hình kết nối tại từng Service**
  - Tạo tệp cấu hình database `src/config/db.ts` tại mỗi microservice.

- [x] **Bước 3: Khởi tạo kết nối độc lập**
  - Sử dụng Mongoose kết nối riêng biệt cho từng service:
    - **`user-service`** kết nối tới `${MONGO_URI}/db_users`
    - **`product-service`** kết nối tới `${MONGO_URI}/db_products`
    - **`order-service`** kết nối tới `${MONGO_URI}/db_orders`

- [x] **Bước 4: Cài đặt bộ theo dõi sự kiện kết nối (Connection Event Listeners)**
  - Lắng nghe các sự kiện `connected`, `error`, và `disconnected` trên từng service.
  - Thiết lập in log tiếng Việt thân thiện, rõ nghĩa ra Console.
  - *Ví dụ:* `console.log('📁 Đã kết nối thành công tới cơ sở dữ liệu db_products!');`
