## 📋 Danh sách các đầu việc thực thi (Checklist)

- [x] **Bước 1: Triển khai các Model Sản phẩm (trong `product-service` - kết nối `db_products`)**
  - Tạo thư mục `src/models/` tại `product-service`.
  - [x] Xây dựng file `Category.ts`: Định nghĩa Interface và Schema cho `CategoryModel`.
  - [x] Xây dựng file `Product.ts`: Định nghĩa Interface và Schema cho `ProductModel`.
  - **LƯU Ý CỨNG:** Tuyệt đối không thêm các trường `rating` và `reviewCount` vào Product Schema.

- [x] **Bước 2: Triển khai Model Tài khoản (trong `user-service` - kết nối `db_users`)**
  - Tạo thư mục `src/models/` tại `user-service`.
  - [x] Xây dựng file `User.ts`: Định nghĩa Interface, Schema cho `UserModel` (hỗ trợ phân quyền `role` và tích lũy điểm `points`).

- [x] **Bước 3: Triển khai các Model Đơn hàng (trong `order-service` - kết nối `db_orders`)**
  - Tạo thư mục `src/models/` tại `order-service`.
  - [x] Xây dựng file `Order.ts`: Định nghĩa Interface và Schema cho `OrderModel` (cấu hình mảng `items`, tổng tiền `totalAmount` và `status` khớp State Machine).
  - [x] Xây dựng file `PrescriptionVault.ts`: Định nghĩa Interface và Schema cho `PrescriptionVaultModel` (cấu hình tự động gia hạn 30 ngày cho `expiryDate` qua pre-save hook).

- [x] **Bước 4: Cấu hình `timestamps: true`**
  - Đảm bảo tất cả các Schemas trên đều có thuộc tính `{ timestamps: true }` để tự động tạo trường `createdAt` và `updatedAt`.
