# 05.2.4 Testing & Validation (Core Validation Verification Checklist)

Checklist hướng dẫn viết và chạy các bộ Unit Test để xác thực các quy tắc ràng buộc y tế cứng (Mongoose Validation) hoạt động ổn định và chính xác 100%.

---

## 📋 Danh sách các đầu việc thực thi (Checklist)

- [x] **Bước 1: Thiết lập thư viện Jest cho kiểm thử**
  - Cài đặt các package cần thiết tại các service:
    ```bash
    npm install --save-dev jest ts-jest @types/jest
    ```
  - Khởi tạo file cấu hình chạy test `jest.config.js`.

- [x] **Bước 2: Viết kịch bản kiểm thử cho `PrescriptionVault` (Đơn thuốc y tế)**
  - Tạo file `tests/prescription.test.ts` trong `order-service`.
  - [x] Kịch bản 1: Tạo mới một bản ghi đơn thuốc y tế chỉ gửi `issueDate`. Kiểm chứng xem hook pre-save có tự động cộng thêm 30 ngày để điền tự động vào trường `expiryDate` hay không.
  - [x] Kịch bản 2: Bắn payload thiếu các trường bắt buộc (`thumbnailUrl`) kiểm chứng Mongoose trả về lỗi Validation Error chuẩn xác.

- [x] **Bước 3: Viết kịch bản kiểm thử cho `Product` (Thuốc & Sản phẩm y tế)**
  - Tạo file `tests/product.test.ts` trong `product-service`.
  - [x] Kịch bản 1: Tạo mới sản phẩm với giá âm hoặc số lượng tồn kho âm để kiểm chứng validation `min: 0` có từ chối bản ghi thành công hay không.
  - [x] Kịch bản 2: Kiểm chứng các trường enum của phân loại thuốc (`OTC` / `Rx`) hoạt động chuẩn chỉnh.

- [x] **Bước 4: Thiết lập lệnh chạy test trong `package.json`**
  - Khai báo script chạy test tại file `package.json` của các service:
    ```json
    "test": "jest"
    ```

---

## 🧪 HOẠT ĐỘNG KIỂM THỬ (Verification Steps)

> [!NOTE]
> ### 🔍 Quy trình Kiểm tra
> 1. Chạy lệnh kiểm thử tự động tại từng service:
>    ```bash
>    npm run test
>    ```
>
> ### 🎯 Tiêu chí Pass
> - 100% các ca kiểm thử (Test cases) đều hiển thị trạng thái `PASSED` màu xanh lục.
> - Hạn sử dụng toa thuốc y tế (`expiryDate`) luôn được gia hạn chính xác 30 ngày kể từ ngày bác sĩ ký đơn (`issueDate`).
